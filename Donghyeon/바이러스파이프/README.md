# [level 2] 바이러스 파이프 - 468373

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/468373)

### 문제 설명

<p><code>1</code>번부터 <code>n</code>번까지 번호가 붙은 <code>n</code>개의 배양체를 <code>n-1</code>개의 파이프로 이어 하나의 트리 모양을 만들었습니다. 각 파이프는 <code>A</code>,<code>B</code>,<code>C</code> 3개의 종류 중 하나로 초기에 모든 파이프는 닫혀있습니다.</p>

<p>배양체 중 하나가 바이러스에 감염되어 있습니다. 바이러스에 감염된 배양체는 열린 파이프를 통해 연결된 다른 인접한 배양체를 감염시킵니다.</p>

<p>당신은 종류가 같은 파이프를 한꺼번에 모두 열었다가 닫을 수 있습니다. 단, 한 종류의 파이프를 연 후 다시 닫기 전에 다른 종류의 파이프를 열 수 없습니다. 파이프를 열었다 닫는 행동을 최대 <code>k</code>번 반복해 최대한 많은 배양체에 바이러스를 감염시키려고 합니다.</p>

<p>배양체의 개수를 나타내는 정수 <code>n</code>, 감염된 배양체의 노드 번호를 나타내는 정수 <code>infection</code>, 파이프의 정보를 나타내는 2차원 정수 배열 <code>edges</code>, 최대 행동 수를 나타내는 정수 <code>k</code>가 매개변수로 주어집니다. 최대 <code>k</code>번 파이프를 열었다 닫은 후, 감염된 배양체 개수의 최댓값을 return 하도록 solution 함수를 완성해 주세요.</p>

## 풀이 방식과 생각의 방향

### 1. 문제를 어떻게 바라봤는가

한 번의 "행동"(파이프 타입 하나를 열었다 닫음)은 `현재 감염 집합 → 다음 감염 집합`으로 바꾸는 상태 전이라고 해석.
이 전이를 최대 `k`번 합성했을 때 나올 수 있는 감염 집합 크기의 최댓값을 구하는 문제로 생각.

`n ≤ 100`, `k ≤ 10`, 파이프 타입 ≤ 3이라는 제약을 보고 최악의 분기 수가 `3^10`(약 6만) 수준이라 판단했고, 별도의 그리디나 DP 점화식을 고민하기보다 **상태(감염 집합)를 그대로 들고 다니는 완전 탐색(DFS)** 으로 접근. 작은 제약은 곧 "정확한 시뮬레이션을 그대로 브루트포스해도 된다"는 신호로 읽음.

### 2. 상태와 전이 설계

- 상태 = `(현재 감염된 노드 집합, 지금까지 사용한 행동 횟수)`
- 전이 = 실제로 존재하는 파이프 타입 중 하나(`t`)를 선택 → 현재 감염된 모든 노드를 동시 시작점으로 하는 다중 시작점 BFS를 타입 `t`의 간선으로만 수행 → 새로 도달한 노드를 감염 집합에 추가
- 이 전이를 DFS로 최대 `k`번 반복하면서, **리프뿐 아니라 재귀에 진입하는 모든 시점**에서 감염 집합 크기의 최댓값을 갱신. `k`번을 다 쓰지 않고 멈추는 것도 항상 가능한 선택이기 때문.

### 3. 가지치기가 핵심이었던 부분

"이번 타입을 열어도 새로 감염되는 노드가 하나도 없다면, 그 이후로 뭘 하든 이 분기는 이미 최댓값에 반영됐다"는 점에 착안해 `nextInfected.size > currentInfected.size`일 때만 재귀를 진행하도록 함.

이 가지치기 덕분에 "같은 타입을 연속으로 열 수 없다"는 조건을 별도 플래그로 관리하지 않아도 됨 — 같은 타입을 다시 열어도 이미 감염된 노드만 다시 도달할 뿐 집합이 늘지 않으므로 자연스럽게 걸러짐. 또한 문제에 실제로 등장하는 타입만 `availableTypes`로 모아서, 등장하지 않는 타입까지 불필요하게 탐색하는 낭비를 줄임.

### 4. 구현 세부사항

- 트리를 무방향 인접 리스트로 구성하고, 각 간선에 타입 정보를 함께 저장.
- 내부 BFS 큐는 `shift()` 대신 인덱스 포인터(`head`)를 사용해 매 단계 O(1) pop이 되도록 함.
- 감염 집합은 `Set`으로 관리해 중복 판별과 삽입을 O(1)에 처리.
