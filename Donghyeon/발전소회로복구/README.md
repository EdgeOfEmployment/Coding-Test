### 문제 설명

<p>대정전이 발생한 발전소는 1~<code>h</code>층으로 이루어진 <code>h</code>층 건물이며, 각 층은 같은 구조의 <code>n</code>×<code>m</code> 크기의 격자로 이루어져 있습니다. 각 칸은 통행 가능한 통로('.'), 통행 불가능한 폐쇄 구역('#'), 엘리베이터('@') 중 하나입니다. 엘리베이터는 격자에서 단 한 칸에만 존재하며 다른 층으로 이동하려면 반드시 엘리베이터를 이용해야 합니다. </p>

<p>통로 중 <code>k</code>곳에는 1~<code>k</code>번의 서로 다른 번호를 가진 회로 패널이 1개씩 설치되어 있습니다. 패널의 위치는 [<code>f</code>, <code>r</code>, <code>c</code>] 형태로 주어지며, <code>f</code>층의 <code>r</code>행 <code>c</code>열에 해당 패널이 위치함을 나타냅니다. 초기에 모든 패널은 비활성화 상태이며, <strong>기술자는 항상 1번 패널이 설치된 위치에서 출발해 모든 회로 패널을 활성화해야 합니다.</strong></p>

<p>기술자는 상하좌우로 인접한 칸으로 1초에 1칸씩 이동할 수 있습니다. 엘리베이터 칸에서만 다른 층으로 이동할 수 있으며, 한 층을 이동할 때마다 1초가 필요합니다. 즉 <code>x</code>층에서 <code>y</code>층으로 이동할 때 <code>|x-y|</code> 만큼 시간이 소요됩니다. 엘리베이터를 기다리는 시간이나 타고 내리는 시간은 고려하지 않습니다.</p>

<p>각 패널은 안전 순서를 따라 활성화해야 합니다. 안전 순서는 여러 개의 [<code>a</code>, <code>b</code>] 쌍으로 이루어져 있으며, <code>b</code>번 패널을 활성화하기 전에 <code>a</code>번 패널이 이미 활성화되어 있어야 함을 나타냅니다.</p>

<p>기술자가 패널이 설치된 칸에 위치할 때 그 패널을 즉시 활성화할 수 있습니다. 단, 안전 순서가 아직 충족되지 않은 패널은 활성화할 수 없습니다. 기술자가 <code>x</code>번 패널을 활성화하기 위해서는 안전 순서에 존재하는 <code>b</code> = <code>x</code>인 모든 [<code>a</code>, <code>b</code>] 쌍에 대해서 모든 <code>a</code>번 패널이 이미 활성화되어 있어야 합니다.</p>

<ul>
<li>예를 들어, 안전 순서 쌍이 [[3, 1], [4, 1], [5, 1], [2, 3]]이라고 가정해 보겠습니다.

<ul>
<li>1번 패널을 활성화하려면 3, 4, 5번 세 패널이 이미 활성화되어 있어야 합니다.</li>
<li>3번 패널을 활성화하려면 2번 패널이 이미 활성화되어 있어야 합니다.</li>
<li>2, 4, 5번 패널은 선행 제약 없이 바로 활성화 가능합니다.</li>
</ul></li>
</ul>

<p>당신은 기술자가 모든 패널을 안전하게 활성화하는 데 필요한 최소 시간을 구해야 합니다.</p>

<p>예를 들어 3층인 발전소 내부 구조가 아래 그림과 같다고 가정해 보겠습니다. 엘리베이터가 5행 2열에 위치해 있습니다.</p>

## 풀이

문제를 보면 결국 "패널 간 이동 비용을 최소화하면서 선행 조건을 지키는 순서로 모든 패널을 방문하는 최단 경로 문제"로 보였다. 다만 각 층의 격자 구조가 모두 동일하다는 점이 특이해서, 이걸 어떻게 활용할지가 첫 번째 고민

같은 층 내에서는 상하좌우 이동만 가능하고 층 이동은 반드시 엘리베이터를 거쳐야 하므로, 두 지점(패널/엘리베이터)의 실제 이동 비용은 "같은 층이면 2D 최단거리, 다른 층이면 (출발지→엘리베이터 2D거리) + (층 차이) + (엘리베이터→도착지 2D거리)"로 항상 분리해서 계산할 수 있음. 격자가 층마다 동일하므로 2D 최단거리는 딱 한 번(각 거점을 시작점으로 BFS)만 구해두면 모든 층에서 재사용할 수 있어, 굳이 3차원 공간 전체를 BFS로 탐색할 필요가 없음.

이렇게 거점 간 이동 비용을 미리 다 구해두고 나면, 문제는 "선행 조건(부분 순서)을 지키면서 k개의 패널을 모두 활성화하는 최소 이동 시간"을 구하는 문제로 좁혀진다. k ≤ 15로 매우 작기 때문에, "현재 위치한 패널 번호 + 지금까지 켜진 패널들의 상태"를 하나의 상태로 보고, 상태 간 이동을 다익스트라로 풀면 된다고 판단
=> 간선 비용(이동 시간)이 모두 0 이상이고, 상태는 켜진 패널 수가 늘어나는 방향으로만 전이되므로 다익스트라 적용할 수 있음

이 아이디어를 정리하여 다음과 같은 순서로 풀이

1. 격자에서 엘리베이터 위치를 찾고, 엘리베이터 + k개 패널을 하나의 거점(POI) 목록으로 묶는다.
2. 각 거점을 시작점으로 BFS를 1번씩 돌려서, 모든 거점 쌍의 2D(같은 층 기준) 최단거리(distMap2D)를 미리 구해둔다.
3. distMap2D를 이용해 패널 u→v로 실제 이동하는 비용(cost)을 계산한다.
    - 같은 층이면 distMap2D[u][v] 그대로 사용한다.
    - 다른 층이면 distMap2D[u][엘리베이터] + |층 차이| + distMap2D[엘리베이터][v]로 계산한다.
4. seqs를 파싱해 각 패널 b를 켜기 위해 필요한 선행 패널 목록(deps[b])을 만든다.
5. 상태를 (현재 위치한 패널 번호, 켜진 패널들을 나타내는 문자열)로 정의하고, 1번 패널 위치에서 시간 0으로 시작해 최소 힙 기반 다익스트라를 수행한다.
    - 다음에 켤 후보 패널은 아직 안 켜져 있고, deps 조건을 모두 만족하는 패널만 고른다.
    - cost[curr][next]만큼 이동 시간을 더해 새로운 상태의 최소 시간을 갱신하고 큐에 넣는다.
6. 켜진 패널 수(count)가 k에 도달하는 순간의 time을 그대로 반환한다(다익스트라이므로 첫 도달이 곧 최소 시간).

생각의 흐름을 요약하면: "이동 비용 계산과 선행 조건을 지키는 방문 순서 결정은 서로 다른 문제이니 나눠서 풀자 → 이동 비용은 격자가 층마다 같다는 점을 이용해 2D BFS 한 번 + 층 차이 계산으로 미리 다 구해둘 수 있다 → 남은 문제는 결국 '상태(현재 위치, 켜진 패널 집합)별 최소 시간'을 구하는 다익스트라다 → k가 작으니 상태 수가 충분히 감당 가능하다"는 순서로 접근

## 다익스트라 알고리즘 정리

다익스트라는 하나의 시작 정점에서 다른 모든 정점까지의 최단 거리를 구하는 알고리즘으로, 간선 가중치가 모두 0 이상일 때만 사용할 수 있다. 매 단계마다 "지금까지 발견된 상태들 중 비용(time)이 가장 작은 상태"를 우선순위 큐(최소 힙)에서 꺼내 확정하고, 그 상태에서 갈 수 있는 다음 상태들의 비용을 더 작게 갱신할 수 있으면 갱신(relaxation)해서 큐에 다시 넣는 과정을 반복한다.

이 문제에서 특이한 점은 다익스트라의 "정점"이 격자의 칸이 아니라 (현재 위치한 패널 번호, 지금까지 켜진 패널들의 집합) 이라는 상태라는 것이다. 즉 일반적인 그래프 위의 최단 거리가 아니라, 상태 공간(state space) 위에서의 최단 거리를 구한 것이다. 이렇게 확장해도 다익스트라가 성립하는 이유는 동일하다.

- cost[curr][next] (패널 간 이동 시간)이 항상 0 이상이므로, 한 번 어떤 상태에 최소 시간으로 도달했다면 그 이후에 그 상태로 더 빠르게 도달할 방법은 없다.
- 코드에서 `if (time > (dist.get(...) || Infinity)) continue;` 부분이, 이미 더 적은 비용으로 확정된 상태를 큐에서 다시 꺼냈을 때 무시하는 처리(lazy deletion)로, 다익스트라 구현에서 흔히 쓰는 패턴이다.
- BFS가 (가중치 없는 그래프에서) 큐에 넣은 순서대로 처리해 첫 방문이 곧 최단 거리가 되는 것처럼, 다익스트라는 우선순위 큐 덕분에 `count === k`(모든 패널이 켜진 상태)를 만족하는 상태를 맨 처음 꺼내는 순간이 곧 전체 문제의 최소 시간이 된다. 그래서 그 시점의 time을 바로 반환할 수 있다.

## 비트마스킹이란

비트마스킹은 정수의 각 비트(0 또는 1)를 이용해 여러 개의 boolean 상태를 하나의 숫자로 압축해서 표현하는 기법이다. k개의 대상이 있고 각각 "켜짐/꺼짐"처럼 두 가지 상태만 가진다면, k비트 정수 하나로 전체 2^k가지 조합을 표현할 수 있다. 예를 들어 k=4일 때 이진수 1011(십진수 11)은 "1, 2, 4번 패널은 켜져 있고 3번 패널은 꺼져 있다"는 뜻이 된다.

자주 쓰는 비트 연산은 다음과 같다.

- i번째가 켜져 있는지 확인: `(state & (1 << (i - 1))) !== 0`
- i번째를 켜기: `state | (1 << (i - 1))`
- 모두 켜졌는지 확인: `state === (1 << k) - 1`

현재 코드는 켜진 패널 상태를 `'0'.repeat(k)` 같은 문자열로 표현하고, 패널을 켤 때마다 `substring`으로 새 문자열을 만들어 갱신하는 방식을 썼다. 이를 비트마스킹으로 바꾸면 대략 다음과 같이 대응된다.

- `const startState = '0'.repeat(k);` → `const startState = 0;`
- `if (stateStr[next - 1] === '1') continue;` → `if ((stateStr >> (next - 1)) & 1) continue;`
- `const nextStateStr = stateStr.substring(0, next - 1) + '1' + stateStr.substring(next);` → `const nextState = stateStr | (1 << (next - 1));`
- `count === k` 종료 조건도 `stateStr === (1 << k) - 1` 로 그대로 대체할 수 있고, 이 경우 count 변수를 따로 들고 다닐 필요도 없어진다.
- `dist` Map의 key로 쓰던 문자열 `` `${curr},${stateStr}` `` 도, `curr * (1 << k) + stateStr` 처럼 정수 하나로 합쳐서 Map 대신 배열(Int32Array 등)로 관리할 수 있다.

비트마스킹으로 바꾸면 좋은 이유는, 문자열 방식은 상태를 갱신할 때마다 `substring`으로 매번 길이 k짜리 새 문자열을 만들어야 하는데(연산량 O(k)), 정수 비트 연산(`&`, `|`, `<<`)은 CPU가 한 번에 처리하는 O(1) 연산이라 훨씬 빠르고, Map 키 비교도 문자열 해시 비교보다 정수 비교가 가볍기 때문이다. k ≤ 15라 상태 수가 최대 2^15 = 32,768개로 충분히 작으므로, curr(최대 15) × state(최대 32,768) 조합을 아예 배열로 미리 잡아두고 Map 없이 O(1) 배열 접근으로 처리하는 것도 가능해서, 현재 927ms인 실행 시간을 더 줄일 여지가 있다.
