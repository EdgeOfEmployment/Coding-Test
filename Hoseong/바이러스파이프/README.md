# 🦠 [프로그래머스] 바이러스 파이프 (2025 카카오 하반기 1차)

## 📌 문제 핵심 요약

### 구조

- `n`개의 노드(배양체)와 `n-1`개의 간선(파이프)으로 구성된 **트리(Tree)** 구조
- 모든 노드는 연결되어 있음

### 파이프 상태

- 각 파이프는 3가지 종류로 구성
  - A 타입 (1)
  - B 타입 (2)
  - C 타입 (3)
- 초기 상태는 모든 파이프가 닫혀 있음

### 행동

- 한 번의 행동에서 하나의 파이프 종류를 선택
- 선택한 종류의 파이프를 열었다가 닫음
- 최대 `k`번까지 반복 가능

### 목표

- 이미 감염된 노드에서 시작
- 현재 열린 파이프를 통해 연결된 노드로 감염 전파
- `k`번의 행동 이후 감염 가능한 배양체 개수의 최댓값 구하기

---

# 💡 알고리즘 접근 방식

## 1. 상태 공간 탐색 (State Space Search)

이 문제는 매 단계마다 3가지 파이프 중 하나를 선택하는 **의사결정 문제**로 볼 수 있습니다.

### 상태(State)

현재 탐색 상태를 다음과 같이 정의합니다.

(step, infected_mask)

| 변수          | 의미                      |
| ------------- | ------------------------- |
| step          | 현재까지 선택한 행동 횟수 |
| infected_mask | 현재 감염된 노드 상태     |

---

### 상태 전이 (Transition)

각 단계마다 선택 가능한 경우는 총 3가지입니다.

A 파이프 선택
B 파이프 선택
C 파이프 선택

따라서 상태 공간 트리의 깊이는 최대 `k`입니다.

---

### 탐색 범위

제한 조건:

k ≤ 10
N ≤ 100

가능한 선택 경로:

3^k = 3^10 = 59,049

상태 수가 매우 작기 때문에 **백트래킹(DFS)** 으로 모든 경우를 탐색합니다.

---

# 2. 감염 전파 알고리즘 (BFS / Flood Fill)

특정 파이프 타입을 선택했을 때 감염되는 과정은 그래프 탐색과 동일합니다.

예)

현재 감염 노드
↓
같은 타입의 파이프 연결
↓
인접 노드 감염
↓
새로운 감염 노드에서 반복

---

## BFS 진행 과정

1. 현재 감염된 모든 노드를 큐에 삽입

2. 큐에서 노드를 하나씩 꺼냄

3. 연결된 인접 노드 확인

4. 조건 만족 시 감염 처리

조건:

현재 파이프 타입 == 선택한 파이프 타입
AND
아직 감염되지 않은 노드

5. 더 이상 감염 가능한 노드가 없을 때 종료

---

# 3. 탐색 최적화 (Pruning)

완전 탐색 과정에서 불필요한 탐색을 줄이기 위해 가지치기를 적용합니다.

## ✅ 무변화 가지치기

특정 파이프를 선택했지만 새로운 감염이 발생하지 않는 경우

next_count == current_count

해당 선택은 의미가 없습니다.

따라서:

현재 경로 종료

하고 이전 상태로 돌아갑니다.

---

## ✅ 조기 종료

이미 모든 노드가 감염된 경우

current_count == N

더 이상 탐색할 필요가 없습니다.

즉시 종료합니다.

---

# 📈 시간 복잡도 분석

## 상태 탐색 개수

최대 탐색 경로:

O(3^k)

k = 10일 경우:

3^10 = 59,049

---

## 상태 전이 비용

한 번의 파이프 선택 후 감염 전파:

BFS 탐색

최대:

O(N)

---

## 전체 시간 복잡도

O(3^k × N)

---

## 최대 입력 기준

3^10 × 100
= 5,904,900

약 590만 번의 연산으로 충분히 처리 가능합니다.

따라서 일반적인 시간 제한(1~2초) 내에서 안정적으로 해결 가능합니다.
