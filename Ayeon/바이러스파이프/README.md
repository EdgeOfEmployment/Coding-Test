# 🦠 바이러스 파이프 풀이 과정 요약

## 1. 문제의 핵심
* **목표:** 최대 $k$번 파이프를 열고 닫아 **최대한 많은 배양체를 감염시키는 것.**
* **특징:** 파이프 종류는 3가지(A, B, C)이며, 한 번에 한 종류만 작동 가능함.
* **핵심 조건:** 이미 감염된 노드와 **'현재 열린 파이프'로 연결된 이웃 노드만** 잉크처럼 번지며 감염됨.

---

## 2. 단계별 알고리즘 흐름

### 1단계: 트리 그래프 만들기 (인접 리스트 구축)
* 배양체들의 연결 상태와 파이프 종류(1, 2, 3)를 양방향 그래프 구조(`adj`)로 저장함.

### 2단계: 모든 파이프 조합 시험하기 (DFS 탐색)
* 매 턴마다 1번을 열지, 2번을 열지, 3번을 열지 3갈래로 나누어 모든 경우의 수를 확인(최대 $k$번 재귀 호출)함.

### 3단계: 바이러스 퍼뜨리기 시뮬레이션 (BFS 탐색)
* 특정 파이프를 열었을 때, **현재 감염된 모든 노드들을 시작점**으로 삼아 파이프를 타고 번지는 노드들을 수집함.
* 아직 감염되지 않았거나, 더 빨리 감염시킬 수 있는 경로를 찾으면 감염 턴수(`turn + 1`)를 갱신함.

### 4단계: 정답 갱신 및 종료
* 매 분기마다 총 감염된 개수를 세어 **최댓값(`max_infected_count`)을 지속적으로 갱신**함.
* 행동 횟수가 $k$번에 도달하면 해당 탐색을 종료함.

---

## 3. 요약 포인트
* **딕셔너리 관리 이유:** 어떤 노드가 **'몇 번째 턴에 일찍 감염되었는지'** 기록해야 남은 턴 동안 더 멀리 퍼뜨릴 수 있기 때문임.
* **풀이 타당성:** 데이터 범위($n \le 100, k \le 10$)가 매우 작아, 모든 경우의 수를 정직하게 다 눌러보는 방식(DFS+BFS)으로도 시간 내에 충분히 통과 가능함.
