# 🌳 트리 분배 노드 최적화 풀이 과정 요약

## 1. 문제의 핵심
* **목표:** 제한 조건(`dist_limit`, `split_limit`) 하에서 만들 수 있는 **리프 노드 수의 최댓값을 구하는 것.**
* **핵심 조건 1:** 모든 리프 노드의 분배도(루트부터 해당 리프까지 부모 자식 수들의 누적 곱)가 `split_limit` 이하여야 함.
* **핵심 조건 2:** 사용 가능한 총 분배 노드의 개수가 `dist_limit` 이하여야 함.
* **구조적 특징:** 같은 깊이에 있는 분배 노드는 모두 자식 수가 같아야 하나, **모든 가지가 동일한 깊이까지 뻗어 나갈 필요는 없음.** (일부 노드는 분배하지 않고 조기에 리프 노드로 확정을 지을 수 있음.)

---

## 2. 알고리즘 설계 및 최적화 흐름

### 1단계: 완전 대칭 트리 기준 탐색 범위 설정
* 분배도는 2자식 분배($x$번)와 3자식 분배($y$번)의 곱($2^x \times 3^y$)으로 결정됨.
* `split_limit` 범위($10^9$) 내에서 가능한 모든 3자식 층 수($y < 23$)와 2자식 층 수($x < 32$)의 조합을 완전 탐색함.
* 가성비가 더 좋은 3자식 분배 층($y$)을 위쪽에 먼저 배치하고, 2자식 분배 층($x$)을 그 아래에 배치하는 구조가 분배 노드 소모를 최소화하는 유일한 최적 구조임.

### 2단계: 완전 대칭 트리 분배 노드 계산
* 3자식 층 $y$개와 2자식 층 $x$개를 꽉 채운 대칭 트리의 필수 분배 노드 수를 수학적으로 연산함.
  * 3자식 분배 노드 수: $\frac{3^y - 1}{2}$
  * 2자식 분배 노드 수: $3^y \times (2^x - 1)$
  * 총합(`full_dist_nodes`): $nodes\_3 + nodes\_2$

### 3단계: 잔여 분배 노드를 활용한 부분 쪼개기 (Case 분기)

#### Case 1: 꽉 채운 트리가 `dist_limit` 이내로 해결되는 경우
* 대칭 트리를 구축하고도 분배 노드 허용량(`dist_limit`)이 남아있다면, 맨 아래 리프 노드 중 일부를 선택하여 추가로 분배함.
* 다음 층을 **2자식 분배 노드로 쪼개는 안(Case A)**과 **3자식 분배 노드로 쪼개는 안(Case B)**의 리프 수 증가분을 각각 시뮬레이션하여 최댓값을 갱신함.
  * 2자식 1개 추가 시: 리프 노드 1개 순증가
  * 3자식 1개 추가 시: 리프 노드 2개 순증가

#### Case 2: 꽉 채우기엔 `dist_limit`이 부족한 경우
* 3자식 층 $y$개는 다 채울 수 있으나 2자식 층 $x$개를 전부 채우지 못하는 경우, 남은 분배 노드(`rem`) 개수만큼만 맨 아래층에서 부분적으로 2자식 분배를 수행하여 리프 수를 늘림.
* 단, 추가 분배 후의 도달 깊이 분배도가 `split_limit` 이하인지 반드시 사전에 검증함.

---

## 3. 요약 포인트
* **그리디(Greedy)적 배치 배분:** 분배 효율을 극대화하기 위해 3자식 분배 노드를 루트 근처(상단 레벨)에 집중 배치하는 구조를 전제함.
* **부분적 분배 허용 처리:** 모든 자식이 완벽하게 똑같이 쪼개지는 대칭 구조만 따지지 않고, **남은 자원을 활용해 일부 가지만 더 뻗어나가는 부분 최적화**를 수학적으로 반영하여 오답을 방어함.
