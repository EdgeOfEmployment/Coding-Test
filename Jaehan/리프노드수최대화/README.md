# 리프 노드 수 최대화 문제 풀이 공유

## 1. 문제 접근법

**상태 정의**

각 깊이에서 `(분배 가능한 노드 수 m, 지금까지의 분배도 p, 남은 분배 노드 예산 D)`를 상태로 두고, 각 깊이마다 자식 수를 2 또는 3 중 선택하는 문제로 정의했습니다.

**핵심 관찰 (브루트포스로 검증)**

- 남은 예산 `D`는 항상 최대한(`min(m, D)`) 소진하는 것이 최적입니다. 분배 노드 1개는 리프 1개를 최소 2개로 바꾸므로 예산을 남기는 것은 항상 손해입니다.
- 깊이별 분배 계수(2 또는 3)의 최적 배열 순서는 항상 **오름차순**입니다. 즉 앞쪽 깊이에서 2를 먼저 쓰고 뒤쪽 깊이에서 3을 쓰는 것이 항상 최적입니다. (2를 먼저 쓰면 예산 소모 속도가 느려져 더 깊고 넓게 분배할 수 있음)

**탐색 축소**

위 두 관찰 덕분에 깊이별로 개별 선택을 탐색할 필요 없이, "2를 몇 번(`c2`) 쓰고 3을 몇 번(`c3`) 쓸지"만 결정하면 됩니다. `split_limit ≤ 10^9`이므로 깊이는 최대 약 30단계로 제한되어, `c2 × c3` 이중 루프(최대 약 30×30)로 전역 최적해를 구할 수 있습니다.

## 2. 해결 코드

```python
def solution(dist_limit, split_limit):
    max_steps = 1
    while 2 ** max_steps <= split_limit:
        max_steps += 1
    max_steps += 2

    def simulate(c2, c3):
        m, p, D, leaves = 1, 1, dist_limit, 0
        for i in range(c2 + c3):
            b = 2 if i < c2 else 3      # 2를 먼저, 3을 나중에
            if D == 0:
                break
            if p * b > split_limit:      # 분배도 초과 방지
                break
            n = min(m, D)                # 가능한 만큼 최대로 분배
            leaves += m - n              # 분배 못한 노드는 리프로 확정
            m = n * b
            p = p * b
            D -= n
        leaves += m
        return leaves

    best = 0
    for c2 in range(max_steps + 1):
        for c3 in range(max_steps - c2 + 1):
            best = max(best, simulate(c2, c3))
    return best
```

## 3. 구현 전략 및 이유

**브루트포스로 성질을 먼저 검증**

`dist_limit`, `split_limit`이 작은 범위에서 모든 분배 순서를 완전탐색하는 코드를 먼저 작성해 정답을 구하고, 그 위에서 "예산 최대 소진"과 "오름차순 배치"라는 두 가설을 각각 수백 개의 랜덤 케이스로 검증했습니다. 단순 그리디(예: 3을 무조건 먼저 사용)는 랜덤 테스트의 약 30%에서 오답이 나왔기 때문에, 반드시 이 검증 과정을 거쳐야 했습니다.

**`m`과 `p`가 동일하다는 점 활용**

예산을 다 쓰지 않은 구간(성장 단계)에서는 분배 가능한 노드 수 `m`과 분배도 `p`가 항상 같은 값으로 함께 증가한다는 점을 이용해, 상태를 단순화하고 `c2`, `c3` 두 변수만으로 전체 경로를 결정할 수 있게 만들었습니다.

**이중 루프로 축소**

깊이별 개별 선택(`2^30` 경우의 수)을 그대로 탐색하면 시간 초과가 나지만, 오름차순 배치가 항상 최적이라는 성질 덕분에 탐색 대상이 `(c2, c3)` 순서쌍(`O(log(split_limit)^2)`)으로 줄어들어 `dist_limit`, `split_limit`이 `10^9`여도 즉시 계산됩니다.
